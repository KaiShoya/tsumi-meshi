# i18n Integration

概要: 本リポジトリは `@nuxtjs/i18n` を採用しています。ランタイムではモジュールが `vue-i18n` を初期化して `useI18n()` を提供します。

重要事項:
- `nuxt.config.ts` で `i18n` 設定（`langDir`, `locales`, `defaultLocale`, `vueI18n`）が管理される。
- テスト実行環境（Vitest）では Nuxt のモジュール初期化が実行されないため、テストセットアップで `vue-i18n` を登録して `useI18n()` を利用可能にしています（`tests/setup.ts` を参照）。

推奨運用:
- 開発/CI 環境では `pnpm install` 後に `pnpm test` を実行して、テスト環境の i18n 登録が確実に行われることを確認する。
- `i18n` の設定変更（ロケールやメッセージファイル追加）を行った場合は、`.agent/docs/qa/i18n-testing.md` を更新し、テストセットアップでの同期手順を記載する。

参照: `nuxt.config.ts`, `i18n/i18n.config.ts`, `tests/setup.ts`
