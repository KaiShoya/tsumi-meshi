## Goal
Deliver the next incremental release focusing on folder management, tag management, recipe checks (history + stats), and search/filter functionality.

## Tasks (feature-scoped)

- **Folders (API + UI + Store + Tests)**
  - [ ] Add Workers API endpoints: `GET /folders`, `POST /folders`, `PUT /folders/:id`, `DELETE /folders/:id`, `GET /folders/hierarchy`
  - [ ] Implement `FoldersRepository` methods and add any necessary DB migrations
  - [ ] Add Pinia data store actions for folders (`fetchAll`, `create`, `update`, `delete`, `getHierarchy`)
  - [ ] Add page UI for folder management and a folder selector component
  - [ ] Unit tests for repository and stores; integration tests for API
  - [ ] Update `.agent/specs/data-model.md` and `repository-api.md`

- **Tags (API + UI + Store + Tests)**
  - [ ] Add Workers API endpoints: `GET /tags`, `POST /tags`, `PUT /tags/:id`, `DELETE /tags/:id`, `POST /tags/find-or-create`
  - [ ] Implement `TagsRepository` methods
  - [ ] Add Pinia data store actions for tags
  - [ ] Add UI for managing tags (list + create + edit) and tag selection component
  - [ ] Tests and docs updates

- **Recipe Checks (history + stats)**
  - [ ] Add API endpoints: `POST /recipes/:id/checks`, `GET /recipes/:id/checks`, `GET /checks/stats`
  - [ ] Implement `RecipeChecksRepository` methods and queries for stats
  - [ ] Add UI to mark a recipe as checked and view history
  - [ ] Add stats view (basic totals/month/week)
  - [ ] Tests + docs updates

- **Search & Filter**
  - [ ] Backend: add search endpoint / query params to `/recipes` for `q`, `folderId`, `tagIds`
  - [ ] Frontend: search input with debounce, folder & tag filters, and integration with page store
  - [ ] Tests for search behavior and edge cases

## Acceptance criteria
- All new endpoints covered by API docs and tests
- UI components accessible and responsive
- Lint and Typecheck pass for all new code
