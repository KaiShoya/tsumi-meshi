# v0.2 ロードマップ（起点: v0.1 完了後）

## 前提
v0.1 系の全タスク（v0.1.4 / v0.1.5 含む）が完了した段階で着手するメジャー次期リリース。主な目的は「分析・拡張性・運用性の向上」と「拡張機能の完成」です。

## 目標（高レベル）
- 高度な分析機能とダッシュボード
- 大量データに耐えるパフォーマンスとバルク操作
- Chrome 拡張を正式サポート（同期・OAuth）
- 運用（バックアップ・移行・監査）機能の実装
- セキュリティ監査とハードニング

## 想定スコープ（候補）
- Analytics: 詳細イベント収集、時系列集計、可視化ウィジェット
- Bulk operations: 複数レシピの一括編集・削除・タグ付け・インポート/エクスポート（CSV）
- Chrome extension: マニフェスト v3, OAuth フロー, シェア/保存ワークフロー
- Data ops: DB エクスポート/インポート、R2 バックアップ、マイグレーションツール
- Security: 依存性アップデート、脆弱性スキャン、外部監査対応ドキュメント
- Enterprise features（オプション）: SSO、組織管理、監査ログ

## 受け入れ基準
- 主要機能ごとに `.agent/specs/` を更新すること
- ユニット/統合/E2E テストと a11y チェックを追加し CI を通すこと
- 必要な infra 変更（R2、Workers、Pages）をドキュメント化し自動化すること

## タイムライン（例）
- フェーズ 1（2-4 週）: Analytics 基礎設計・イベント収集・ダッシュボード MVP
- フェーズ 2（3-6 週）: Bulk operations + import/export、E2E 堅牢化
- フェーズ 3（2-4 週）: Chrome extension 完成、セキュリティ監査

## 次のアクション
1. v0.1.4 / v0.1.5 の完了基準を確定し、完了後に v0.2 のスコープを確定する
2. v0.2 の主要項目を Issue 化して優先度付けする
3. 重要な非機能要件（データ保持、パフォーマンス目標、SLO）を定義する
